# Docker Project Builder - Web UI

A visual interface for creating and managing Docker projects generated by Docker Project Builder.

## Features

### 1. Project Creation Wizard
- Visual form for selecting services (Next.js, Express, PostgreSQL, Redis, etc.)
- Multi-environment configuration (local, staging, prod)
- Real-time validation
- Executes `dpb create` command with selected options

### 2. Docker Viewer
- View all generated projects
- Control Docker containers (start, stop, restart)
- Real-time logs viewer
- Container status monitoring
- Executes generated bash CLI commands (e.g., `./myapp up`, `./myapp logs`)

### 3. Test Project Feature
- Generate test project with selected services
- Automatically build and start Docker containers
- Curl test endpoints (Next.js app, Express API)
- View detailed logs with copy-to-clipboard
- Validate that generated projects work correctly

## Tech Stack

- **Framework**: Next.js 16 + React 19
- **UI Library**: shadcn/ui (Radix UI + Tailwind CSS)
- **Styling**: Tailwind CSS
- **Icons**: Lucide React
- **Type Safety**: TypeScript

## Getting Started

### Prerequisites

- Node.js 20+ installed
- Docker Desktop running
- `dpb` CLI built and available in parent directory

### Installation

```bash
cd webui
npm install
```

### Development

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) in your browser.

### Build for Production

```bash
npm run build
npm start
```

## Architecture

The web UI is a **thin wrapper** around the `dpb` CLI tool and generated bash scripts:

```
Web UI (Next.js)
    ↓
API Routes (Next.js Server Actions)
    ↓
Execute Shell Commands
    ↓
    ├─ dpb create    (generate projects)
    ├─ dpb list      (list services)
    └─ ./myapp       (Docker operations)
```

**Key Design Principles:**

1. **No Docker Logic in TypeScript**: All Docker operations execute the generated bash CLI
2. **CLI Commands as Source of Truth**: Web UI simply wraps command execution
3. **Project-Scoped Operations**: All operations limited to generated project directories
4. **Local-First**: Runs entirely on localhost, no cloud backend

## API Routes

### `/api/services` (GET)
Lists available plugins by executing `dpb list`

### `/api/generate` (POST)
Generates a new project by executing `dpb create` with user selections

**Request:**
```json
{
  "projectName": "my-app",
  "domain": "my-app.local",
  "services": ["nextjs", "expressjs", "postgres"],
  "environments": ["local", "staging"]
}
```

### `/api/docker/projects` (GET)
Lists all generated projects (directories ending with `-output`)

### `/api/docker/[action]` (POST)
Executes Docker commands on a project's bash CLI

**Actions:**
- `up` - Start containers
- `down` - Stop containers
- `restart` - Restart containers
- `logs` - View logs
- `status` - Container status
- `build` - Build images

**Request:**
```json
{
  "projectName": "my-app",
  "service": "nextjs" // optional
}
```

### `/api/test` (POST)
Runs full integration test:
1. Generate test project
2. Build Docker images
3. Start containers
4. Curl test endpoints
5. Return results and logs

**Request:**
```json
{
  "projectName": "test-project",
  "services": ["nextjs", "expressjs", "postgres"]
}
```

## Development Notes

- Generated projects are created as `{name}-output` directories in the parent folder
- Test projects are automatically added to `.gitignore`
- Web UI uses the `dpb` CLI from `../dpb` (relative to webui directory)
- All Docker operations execute through generated bash CLIs (e.g., `./myapp up`)
- Logs can be copied to clipboard for easy sharing with support
